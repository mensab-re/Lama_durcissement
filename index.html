Nginx hardening

La configuration par défaut du service web Nginx ne suffit pas, il existe des paramètres qui vont nous permettre de le rendre plus sécurisé.

Nous allons principalement modifie ces 2 fichiers :

/etc/nginx/nginx.conf

/etc/nginx/sites-available/alpa

Modification du headers

Dans /etc/nginx/nginx.conf nous allons dans un premier temps modifier ou supprimer le headers renvoyer par le serveur, cela permet de cacher à l'attaquant la version du serveur Nginx.

Le paquet nginx-extras est necessaire.

sudo apt-get install nginx-extras
# /etc/nginx/nginx.conf
......
http {
......
        #### mensab ########################################

        ## modification du headers renvoye par defaut 
        more_set_headers 'Server: Toha Heavy Industries';
......
}

Désactiver les protocols SSL/TLS faibles

N'autoriser que les protocoles TLSv1.2 et 1.3.

    # /etc/nginx/nginx.conf
......
http{
......
    # SSL Settings
    ##

    ssl_protocols TLSv1.2 TLSv1.3; # Dropping SSLv3, ref: POODLE
    ssl_prefer_server_ciphers on;
.....
}

Modification des ressources allouées

Ajouter ces paramettres imposent une taille de buffer pour les clients et permettent de prevenir en partie les attaques DoS ou buffer overflow. 

     # /etc/nginx/nginx.conf
......
http{
......
    ##### buffers settings ####

    client_body_buffer_size 1K;
    client_header_buffer_size 1k;
    client_max_body_size 1k;
    large_client_header_buffers 2 1k;
 .....
}

Désactiver les methods HTTP inutiles

Dans le fichier /etc/nginx/sites-available/alpa.

server {
........................
######## limit http methods #########
    location / {
        ##### http methods ###
        limit_except GET POST { deny  all; }

        try_files $uri $uri/ /index.php$is_args$args;
    }
#####################################################
.......................
}

Ajout des HTTP security headers

     # /etc/nginx/nginx.conf
......
http{
......        
        ##### content security policy ###
        # Limite les attaques XSS et code injection
        add_header Content-Security-Policy "default-src 'self';";

        ##### X-Frame-Options ###
        # limite les attaques clickjacking
        add_header X-Frame-Options "DENY";

        ##### X-XSS-Protection ###
        # limite les attaque XSS
        add_header X-XSS-Protection "1; mode=block";

        ##### X-Content-Type-Options ###
        # limite le MIME type sniffing
        add_header X-Content-Type-Options nosniff;
......
}

Apparmor

Nous pouvons avec Apparmorl imiter les services aux actions qu'il sont censé réaliser, toutes les autres actions que nous n’aurons pas autoriser seront bloquées.

Il faut installer les paquets suivants : apparmor, apparmor-profiles, apparmor-utils.

Verifiez que apparmor soit activer :

cat /sys/module/apparmor/parameters/enabled

La commande aa-status permet de lister les profiles apparmor des applications et des processus ainsi qu'un status sur leur mode.

Nous avons activé les profils fourni avec Apparmor avec la commande aa-enforce /etc/apparmor.d/* et avons créé un profil pour le service Nginx.

aa-autodep nginx génère un profil vide pour nginx

aa-complain nginx met nginx en mode complain qui log les actions mais ne les bloques pas, cela nous permettra dans un premier temps de faire fonctionner nginx et de logger toutes les actions qu'ils est censé faire.

Puis avec la commande aa-logprof nous pouvons choisir d'autoriser ou non ces actions :

Recommandation sur sysctl de l'anssi

Voici la configuration recommandées par l'anssi et applique sur notre serveur :

####################################################### Reseau ##################################

# Pas de routage entre les interfaces
net.ipv4.ip_forward = 0

# Filtrage par chemin inverse
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1

# Ne pas envoyer de redirections ICMP
net.ipv4.conf.all.send_redirects = 0
net.ipv4.conf.default.send_redirects = 0

# Refuser les paquets de source routing
net.ipv4.conf.all.accept_source_route = 0
net.ipv4.conf.default.accept_source_route = 0

# Ne pas accepter les ICMP de type redirect
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.all.secure_redirects = 0
net.ipv4.conf.default.accept_redirects = 0
net.ipv4.conf.default.secure_redirects = 0

# Loguer les paquets ayant des IPs anormales
net.ipv4.conf.all.log_martians = 1

# RFC 1337
net.ipv4.tcp_rfc1337 = 1

# Ignorer les réponses non conformes à la RFC 1122
net.ipv4.icmp_ignore_bogus_error_responses = 1

# Augmenter la plage pour les ports éphémères
net.ipv4.ip_local_port_range = 32768 65535

# Utiliser les SYN cookies
net.ipv4.tcp_syncookies = 1

# Désactiver le support des "router solicitations"
net.ipv6.conf.all.router_solicitations = 0
net.ipv6.conf.default.router_solicitations = 0

# Ne pas accepter les "router preferences" par "router advertisements"
net.ipv6.conf.all.accept_ra_rtr_pref = 0
net.ipv6.conf.default.accept_ra_rtr_pref = 0

# Pas de configuration auto des prefix par "router advertisements"
net.ipv6.conf.all.accept_ra_pinfo = 0
net.ipv6.conf.default.accept_ra_pinfo = 0

# Pas d'apprentissage du routeur par défaut par "router advertisements"
net.ipv6.conf.all.accept_ra_defrtr = 0
net.ipv6.conf.default.accept_ra_defrtr = 0

# Pas de configuration auto des adresses à partir des "router advertisements"
net.ipv6.conf.all.autoconf = 0
net.ipv6.conf.default.autoconf = 0

# Ne pas accepter les ICMP de type redirect
net.ipv6.conf.all.accept_redirects = 0
net.ipv6.conf.default.accept_redirects = 0

# Refuser les packets de source routing
net.ipv6.conf.all.accept_source_route = 0
net.ipv6.conf.default.accept_source_route = 0

# Nombre maximal d'adresses autoconfigurées par interface
net.ipv6.conf.all.max_addresses = 1
net.ipv6.conf.default.max_addresses = 1


####################################################### System ##################################

# Désactivation des SysReq
kernel.sysrq = 0

# Pas de core dump des exécutables setuid
fs.suid_dumpable = 0

# Interdiction de déréférencer des liens vers des fichiers dont
# l'utilisateur courant n'est pas le propriétaire
# Peut empêcher certains programmes de fonctionner correctement
fs.protected_symlinks = 1
fs.protected_hardlinks = 1

# Activation de l'ASLR
kernel.randomize_va_space = 2

# Interdiction de mapper de la mémoire dans les adresses basses (0)
vm.mmap_min_addr = 65536

# Espace de choix plus grand pour les valeurs de PID
kernel.pid_max = 65536

# Obfuscation des adresses mémoire kernel
kernel.kptr_restrict = 1

# Restriction d'accès au buffer dmesg
kernel.dmesg_restrict = 1

# Restreint l'utilisation du sous système perf
kernel.perf_event_paranoid = 2
kernel.perf_event_max_sample_rate = 1
kernel.perf_cpu_time_max_percent = 1
